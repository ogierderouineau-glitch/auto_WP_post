1) What it does

OpenAI supports generating or editing images, and for your case the dedicated endpoint is POST /images/edits. It creates an edited image from one or more input images plus a prompt. The edit endpoint supports GPT image models such as gpt-image-1.5, gpt-image-1, gpt-image-1-mini, and chatgpt-image-latest.【turn922970view1†L720-L724】【turn276343view0†L924-L929】

the original image,

3) The key parameters you’d use

For image editing, the main parameters are:

images: input image references; GPT image models can take up to 16 input images.
prompt: text description of the desired edit.
input_fidelity: high or low.
mask: optional; lets you target only certain areas.
model: e.g. gpt-image-1.5.
n: number of edited outputs, minimum 1, maximum 10.
output_format: png, jpeg, or webp.
quality: low, medium, high, or auto.
size: 1024x1024, 1536x1024, 1024x1536, or auto.【turn276343view0†L931-L1088】

5) Very practical example with curl

The docs show the edit endpoint as a multipart upload using image[] plus a prompt. Their example posts to /v1/images/edits, includes one or more image files, and decodes the returned base64 into a local image file.【turn378314view0†L1236-L1245】

For your use case, a practical version would look like this:

curl -s \
  -X POST "https://api.openai.com/v1/images/edits" \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -F "model=gpt-image-1.5" \
  -F "image[]=@original.jpg" \
  -F 'prompt=Retouch this photo realistically. Preserve the person’s exact face, identity, facial features, skin tone, and overall pose. Do not make the person appear to be smoking.   -F "input_fidelity=high" \
  -F "output_format=png" \
  -F "quality=high" \
  -F "n=1"

6) Node.js example that saves a PNG

If your app receives a WhatsApp image and you want a file back for Codex / VS Code, a plain Node workflow is easy:

import fs from "fs";
import FormData from "form-data";
import fetch from "node-fetch";

async function editPhoto() {
  const form = new FormData();
  form.append("model", "gpt-image-1.5");
  form.append("image[]", fs.createReadStream("./original.jpg"));
  form.append(
    "prompt",
    "Retouch this photo realistically. Preserve the person's exact face, identity, facial features, and pose. Remove cigarette and smoke. Remove bicycle, ashtray, chair, cables, and clutter. Improve lighting and framing. Keep it realistic."
  );
  form.append("input_fidelity", "high");
  form.append("output_format", "png");
  form.append("quality", "high");
  form.append("n", "1");

  const res = await fetch("https://api.openai.com/v1/images/edits", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.OPENAI_API_KEY}`
    },
    body: form
  });

  const json = await res.json();

  const b64 = json.data?.[0]?.b64_json;
  if (!b64) throw new Error("No image returned");

  fs.writeFileSync("edited.png", Buffer.from(b64, "base64"));
  console.log("Saved edited.png");
}

editPhoto().catch(console.error);

That matches the documented shape of the endpoint: image inputs, prompt, optional fidelity/mask/settings, and image data returned in data[].b64_json for GPT image models.【turn276343view0†L931-L949】【turn276343view0†L966-L1088】【turn276343view0†L1115-L1121】

7) When to use a mask

A mask is optional, but for your specific case it is often the best move. The API accepts a mask image as well as the main input image.【turn276343view0†L976-L989】

8) How to prompt for face preservation

For identity-sensitive edits, be very explicit.

A good prompt pattern is:

Retouch this photo realistically. Preserve the subject’s exact face, identity, facial proportions, skin tone, and expression. Do not alter the face. Remove the cigarette and smoke. Remove the bicycle, ashtray, chair, cables, and clutter. Improve lighting and framing. Keep the image natural and photographic.
